# FIFA ワールドカップ26 を振り返るサイト — セットアップ

ポップ／ステッカー風のデザインで、試合結果・順位・スタッツを楽しく振り返るファンサイトです。
Netlify にそのままアップロードして公開できます。

---

## 1. ファイル構成

```
WorldCup26.dc.html   … サイト本体（このファイルを開けば動きます）
support.js           … 表示に必要なランタイム（自動生成・編集不要）
data/wc26.json       … 生データ（試合結果・日程・得点者）。あればこれを読み込み、無ければ内蔵データで表示
tools/update-data.mjs        … APIから生データを取得して wc26.json を書き換えるスクリプト
.github/workflows/update-data.yml … 上記を定期実行する GitHub Actions
```

> **公開用に index.html が必要な場合**は、`WorldCup26.dc.html` を `index.html` という名前でコピーすればトップページになります（`support.js` と `data/` は同じ階層に置いてください）。

---

## 2. Netlify で公開する

1. このフォルダごと GitHub リポジトリにプッシュ
2. Netlify で「Add new site → Import an existing project」からリポジトリを選択
3. ビルドコマンドは空、Publish directory はこのフォルダ（ルート）でOK
4. デプロイ完了 → `https://◯◯◯.netlify.app` で公開

ドラッグ＆ドロップ（Netlify Drop）でフォルダを投げ込むだけでも公開できます。

---

## 3. データの「自動更新」（手動更新はゼロ）

このサイトは **生データ（試合結果・日程・得点者）だけ** を更新すれば成立します。
試合レポート・今日の注目・勝敗予想アンケート・スタッツコラムは、
**ページ側がその生データから毎回自動生成**します（しかも全7言語で）。手書き原稿はありません。

```
サッカーAPI ─(取得)─▶ tools/update-data.mjs ─(生データ書換)─▶ data/wc26.json
                                                                  │
GitHub Actions が定期実行（cron）── コミット ──▶ Netlify が自動で再デプロイ
                                                                  │
              サイトが生データを読み込み、記事・注目・アンケートを自動生成して表示
```

### 何が自動で生まれ変わるか
- **試合レポート**：スコア差・合計得点などからタグ（最大スコア／打ち合い／完封／スコアレス 等）と本文を自動生成
- **今日の注目／見どころ**：FIXTURES（日程）から注目カードを自動選定
- **勝敗予想アンケート**：優勝予想＝順位上位4チーム、次節カード＝直近の試合を自動セット
- **スタッツコラム**：最大スコア差・完封試合数・最多得点チームをデータから自動算出
- **会場名**：日程に会場が含まれていれば、今日の注目・見どころに会場名を表示
- **得点者**：試合データに得点者（選手名・時間）が含まれていれば、試合カードとレポートに自動表示
  し、得点ランキングも実データから集計（worldcup26.ir はライブ期間中に得点者を配信）

### 設定手順（worldcup26.ir／無料・支払い不要）
1. **worldcup26.ir で無料登録してトークンを取得**
   - `POST https://worldcup26.ir/auth/register` に name / email / password を送ると `token` が返ります（84日有効）。
   - 例: `curl -X POST https://worldcup26.ir/auth/register -H "Content-Type: application/json" -d '{"name":"me","email":"you@example.com","password":"****"}'`
2. GitHub リポジトリの **Settings → Secrets and variables → Actions** に、次のどちらかを登録
   - `WC26_TOKEN`（取得したトークン）  ← おすすめ
   - もしくは `WC26_EMAIL` と `WC26_PASSWORD`（実行のたびにログインしてトークン取得）
3. これで `.github/workflows/update-data.yml` の cron（初期値は3時間ごと）に従って
   `data/wc26.json` が自動更新され、サイトに反映されます

> 別のAPI（football-data.org / API-Football など）を使う場合は、`tools/update-data.mjs` の
> エンドポイントURLと変換ロジックを差し替えるだけでOKです。
> 試合が立て込む期間は cron を `*/30 * * * *`（30分ごと）などに上げるとライブ感が出ます。
> 手元で試すには `WC26_EMAIL=... WC26_PASSWORD=... node tools/update-data.mjs` を実行してください。

### 自動更新を使わない場合
`data/wc26.json` の生データ（GROUPS／FIXTURES／SCORERS）を直接書き換えてプッシュすれば、
それだけで記事・注目・アンケートまで丸ごと更新されます。原稿を書く必要はありません。

## 4. 機能メモ

- **スコアティッカー**：上部を流れる全試合のスコア
- **カウントアップ**：総ゴール数などの数字がアニメーション
- **勝敗予想の投票**：クリックで投票、結果は端末（localStorage）に保存
- **データチャート**：得点ランキングのバー、グループ別総得点の棒グラフ（スクロールで伸びる）
- **グループ別タブ**：A〜L を切り替えて順位表と試合結果を表示

- **多言語対応（7言語）**：日本語・English・Español・Português・Deutsch・한국어・العربية を右上で切り替え。
  選択は端末に保存されます。アラビア語は RTL（右→左）レイアウトに自動対応。
  国名は `Intl.DisplayNames` で各言語に自動ローカライズされます（スコットランド/イングランドのみ個別指定）。
  ※ 自動生成される記事は事実ベースのテンプレート方式のため、全言語で自然な文章になります。
  （得点者名は API 由来のラテン表記でそのまま表示します。）

### 表示の切り替え（Tweaks）
エディタ上では以下のオン/オフを切り替えられます。
`showPredictions`（投票）・`showCharts`（チャート）・`liveTicker`（ティッカーの動き）
